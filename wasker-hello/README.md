# Wasker Hello on Unikraft

Build and run an ELF object generated by Wasker on Unikraft, without embedding WAMR.
This example links your `wasm.o` (from Wasker) with a small WASI wrapper implemented for Unikraft userspace.

## What This Example Contains

- `wasi_wrapper.c`: minimal wrappers for common WASI preview1 calls used by simple programs (`fd_write`, `environ_get`, `environ_sizes_get`, `proc_exit`) and linear memory helpers (`memory_base`, `memory_grow`).
- `Makefile.uk`: links `wasi_wrapper.c` and `wasm.o` into the Unikraft image.

## Requirements

- Unikraft build dependencies from [../README.md](../README.md#requirements)
- `wasker` installed on your host
- An input Wasm binary (`.wasm` or `.wat`) that Wasker can compile

## 1. Generate `wasm.o` with Wasker

From this directory:

```console
# Example: compile your wasm module to wasm.o and wasm.ll
wasker /path/to/your/module.wasm
# or
wasker /path/to/your/module.wat

# Place/copy the generated object in this directory as wasm.o
cp ./wasm.o /home/pedro/catalog-core/wasker-hello/wasm.o
```

`make` will fail with a clear error if `wasm.o` is missing.

## 2. Set Up Repositories

```console
./setup.sh
```

Or run the top-level setup once:

```console
cd ..
./setup.sh
```

## 3. Configure and Build

```console
make distclean
wget -O /tmp/defconfig https://raw.githubusercontent.com/unikraft/catalog-core/refs/heads/main/wasker-hello/.scripts/defconfig/qemu.x86_64
UK_DEFCONFIG=/tmp/defconfig make defconfig
make -j $(nproc)
```

## 4. Run

### QEMU/x86_64

```console
qemu-system-x86_64 -nographic -m 32 -cpu max -kernel workdir/build/wasker-hello_qemu-x86_64
```

### QEMU/ARM64

```console
qemu-system-aarch64 -nographic -machine virt -m 32 -cpu max -kernel workdir/build/wasker-hello_qemu-arm64
```

### Firecracker/x86_64

```console
rm -f firecracker.socket
firecracker-x86_64 --config-file fc.x86_64.json --api-sock firecracker.socket
```

### Firecracker/ARM64

```console
rm -f firecracker.socket
firecracker-aarch64 --config-file fc.arm64.json --api-sock firecracker.socket
```

### Xen/x86_64

```console
sudo xl create -c xen.x86_64.cfg
```

### Xen/ARM64

```console
sudo xl create -c xen.arm64.cfg
```

## Notes

- This example does not embed WAMR.
- It expects Wasker output object symbols like `wasker_main` and WASI imports (`fd_write`, `environ_get`, `environ_sizes_get`, `proc_exit`).
- Extend `wasi_wrapper.c` with additional WASI calls as needed by your Wasm program.
